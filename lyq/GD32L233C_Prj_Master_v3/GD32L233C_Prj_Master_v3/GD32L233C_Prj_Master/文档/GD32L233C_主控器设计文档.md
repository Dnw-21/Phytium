# GD32L233C 主控器（Master Controller）设计文档

## 1. 概述

本文档描述基于 **GD32L233C + FreeRTOS** 的主控器软件设计。主控器通过 LoRa 无线模块（ATK-MWCC68D）与最多 **3 个终端节点**通信，接收终端上报的 **6 种数据类型**，将状态数据和故障波形数据分别存储到芯片内部 Flash 的不同区域，并根据节点故障状态向终端下发指令。

- **接收**: 终端上报的 6 种数据类型（状态头 / 波形头 / 电源 / 节点原始数据 / 波形原始数据 / 故障列表）
- **存储**: 状态数据存于 Flash 状态区（`0x08030000`，每节点 20KB），波形数据存于 Flash 波形区（`0x08040000`，每节点 20KB）
- **轮询**: `master_poll_task` 逐节点条件轮询 — 有故障快照则上传故障数据，否则上传正常数据
- **指令下发**: 混沌加密后通过 LoRa 发送，带 ACK 确认 + 最多 3 次重发

## 3. 任务说明

### 3.1 master_recv_task — 主控接收任务

**优先级**: 5（最高） | **堆栈**: 256 字 | **文件**: `app/task/master_recv.c`

从 LoRa 模块接收原始帧数据，解析出 `sync_code` + `data_type` + 加密载荷，打包为 `RecvPacket_t` 后通过 `g_recv_queue` 发送给 `master_process_task` 处理。本身不做解密和分发。

**执行流程**:
```
while(1) {
    1. atk_mwcc68d_receive_data(lora_buf, &recv_len, 5000ms)
       超时或无数据 → continue
    2. 提取 sync_code(8B) + enc_type(1B) + 加密载荷
    3. 打包为 RecvPacket_t → xQueueSend(g_recv_queue)
    4. vTaskDelay(10ms)
}
```

---

### 3.2 master_judge_task — 主控故障判断任务

**优先级**: 3 | **堆栈**: 256 字 | **文件**: `app/task/master_judge.c`

**周期**: 1000ms（每秒轮询所有节点）

**功能**:
- **离线检测**: 节点超过 15 秒无数据 → 标记 `is_online=0`
- **故障触发**: 节点在线且 `severity >= SEVERITY_WARNING`, `fault_type != FAULT_NONE` → 下发 `CMD_REQUEST_WAVEFORM` (0x10) 请求波形

**执行流程**:
```
while(1) {
    vTaskDelay(1000ms)

    for (i = 0; i < MASTER_MAX_NODES; i++) {
        if (now - node.last_recv_time > 15000ms)
            → 标记离线, log_warn, continue

        if (!在线 || fault_type==NONE) → continue
        if (severity < WARNING) → continue

        send_lora_cmd(i, CMD_REQUEST_WAVEFORM, params)
    }
}
```

### 3.3 master_process_task — 主控数据处理任务

**优先级**: 4 | **堆栈**: 512 字 | **文件**: `app/task/master_recv.c`

从 `g_recv_queue` 取出 `RecvPacket_t`，执行**混沌解密**后根据 `DataType_t` 分发到对应处理器。内部维护一个**接收状态机**用于连续累积多包数据。

**接收数据类型及处理方式**:

| 类型 | 枚举值 | 载荷格式 | 处理方式 |
|------|--------|---------|---------|
| `DATA_TYPE_STATUS` | 0x01 | `NodeUploadData_t`(11B) 或 `FaultUploadHeader_t`(15B) | 区分周期上传/故障上传，更新节点快照，标记 is_online, fault_pending |
| `DATA_TYPE_WAVE` | 0x02 | `WaveChunkHeader_t`(12B) | 擦除该节点波形 Flash 区，状态机进入 `RECV_WAVE` |
| `DATA_TYPE_POWER` | 0x03 | 预留 | 仅日志记录 |
| `DATA_TYPE_NODE_RAW` | 0x04 | `NodeSample_t` × N | 累存到 `g_dl_buf.node_buffer[]`，满后整体写入 Flash 状态区 |
| `DATA_TYPE_FLASH_WAVE` | 0x05 | `int16_t` × N | 逐包追加写入 Flash 波形区 |
| `DATA_TYPE_FAULT_LIST` | 0x06 | `uint8_t` 故障有效性数组 | 统计有效故障数，日志输出 |

**状态机流程**:
```
IDLE ──STATUS(0x01)──→ 解析头, 设置 dl 参数, dl->active=1
 IDLE ──WAVE(0x02)────→ 解析头, 擦除旧波形, dl->active=1, state=RECV_WAVE
 IDLE ──其他 ─────────→ 单包处理完成

NODE_RAW(0x04) ──→ 累积到 dl->node_buffer
    满预期点数 → master_flash_save_node_data() → 写 Flash → dl->active=0 → IDLE

RECV_WAVE ──FLASH_WAVE(0x05)──→ 直接写 Flash 波形区
    满 sample_count → 写 Flash → dl->active=0 → IDLE
```

**执行流程**:
```
while(1) {
    1. xQueueReceive(g_recv_queue, &pkt, portMAX_DELAY)
    2. 混沌解密: chaos_decrypt_packet() → data_type + 载荷
    3. 根据 node_id 标记节点在线, 更新时间戳
    4. switch(data_type) 分派到 process_xxx()
}
```

### 3.4 master_poll_task — 主控轮询调度任务

**优先级**: 2 | **堆栈**: 512 字 | **文件**: `app/task/master_poll_task.c`

**周期**: 默认 5000ms（`MASTER_POLL_CYCLE_MS`）

逐节点条件轮询，根据 `fault_pending` 标志决定上传内容：

- **fault_pending == 1**（有待上传故障快照）→ 下发 `CMD_REQUEST_FAULT_DATA` (0x15)，接收故障数据，写入 Flash，清除 `fault_pending`。**不上传正常节点数据**。
- **fault_pending == 0**（无故障快照）→ 下发 `CMD_POLL_STATUS` (0x14)，带时间戳参数，接收正常节点数据。

**执行流程**:
```
while(1) {
    计算轮询时间戳 poll_ts

    for (i = 0; i < MASTER_POLL_MAX_NODES; i++) {
        if (node[i].fault_pending) {
            // 存在故障快照: 下发 CMD_REQUEST_FAULT_DATA
            send_lora_cmd(i, 0x15, [node_id])
            等待下载完成 (TIER2_TIMEOUT_MS = 6000ms)
            等待 Flash 保存完成
            node[i].fault_pending = 0
        } else {
            // 无故障快照: 下发 CMD_POLL_STATUS
            send_lora_cmd(i, 0x14, [poll_ts])
            等待下载完成 (TIER1_TIMEOUT_MS = 3000ms)
        }
    }

    等待剩余周期时间: MASTER_POLL_CYCLE_MS - elapsed
}
```

**LoRa 命令发送**（`send_lora_cmd`，同文件内静态函数）:
```
send_lora_cmd(node_id, cmd_code, params, param_len):
    1. 清空 dl->recv_started / recv_raw_points
    2. 构建 payload[cmd_code | params]
    3. 混沌加密: chaos_encrypt_packet()
    4. LoRa 发送: send_node_data_with_ack()
       最多重试 3 次，等待 ACK
```

## 4. 数据结构

### 4.1 数据类型枚举 (`DataType_t`) — `app/data_frame.h`

```c
typedef enum {
    DATA_TYPE_STATUS     = 0x01,   // 节点状态头 (NodeUploadData_t / FaultUploadHeader_t)
    DATA_TYPE_WAVE       = 0x02,   // 波形数据头 (WaveChunkHeader_t)
    DATA_TYPE_POWER      = 0x03,   // 电源电压数据（预留）
    DATA_TYPE_NODE_RAW   = 0x04,   // 节点原始数据包 (int32×4, 来自环缓冲)
    DATA_TYPE_FLASH_WAVE = 0x05,   // 波形原始数据包 (int16, 来自Flash存储)
    DATA_TYPE_FAULT_LIST = 0x06    // 故障记录列表
} DataType_t;
```

### 4.2 节点采样数据 (`NodeSample_t`) — `app/data_frame.h`

```c
typedef struct {
    int32_t active_power;      // 有功功率 ×10000
    int32_t reactive_power;    // 无功功率 ×10000
    int32_t voltage_angle;     // 电压相角 ×10000
    int32_t voltage_mag;       // 电压幅值 ×10000
} NodeSample_t;                // 共 16 字节
```

### 4.3 故障类型 (`FaultType_t`) — `app/data_frame.h`

```c
typedef enum {
    FAULT_NONE = 0,
    FAULT_OVER_VOLTAGE,      // 过压: Vrms > 1.05pu
    FAULT_UNDER_VOLTAGE,     // 欠压: Vrms < 0.95pu
    FAULT_VOLTAGE_SAG,       // 暂降: Vrms < 0.90pu
    FAULT_VOLTAGE_SWELL,     // 暂升: Vrms > 1.10pu
    FAULT_TRANSIENT          // 暂态突变（预留）
} FaultType_t;
```

### 4.4 故障级别 (`SeverityLevel_t`) — `app/data_frame.h`

```c
typedef enum {
    SEVERITY_NORMAL  = 0,
    SEVERITY_WARNING = 1,
    SEVERITY_DANGER  = 2
} SeverityLevel_t;
```

### 4.5 周期上传头 (`NodeUploadData_t`) — `app/data_frame.h`

```c
typedef struct {
    uint8_t  data_type;         // DATA_TYPE_STATUS

### 4.6 故障上传头 (`FaultUploadHeader_t`) — `app/data_frame.h`

typedef struct {

    uint8_t      severity;      // 故障级别

    FaultType_t  fault_type;    // 故障类型
    uint8_t      node_index;    // 节点号

```

### 4.7 波形数据头 (`WaveChunkHeader_t`) — `app/data_frame.h`

### 4.8 节点信息快照 (`MasterNodeInfo_t`) — `app/task/master.h`

运行时常驻 RAM 的轻量级节点索引（每节点约 30B，3 节点共约 90B）：

```c
typedef struct {
    uint8_t      node_id;                // 节点号 0~2
    uint8_t      is_online;              // 在线标志
    uint8_t      has_status_data;        // Flash 中是否有有效状态数据
    uint8_t      severity;               // 当前故障级别
    FaultType_t  fault_type;             // 当前故障类型
    uint8_t      fault_pending;          // 是否有待上传的故障快照 (1=有)
    uint32_t     last_recv_time;         // 最后通信时间戳
    uint32_t     fault_count;            // 累计故障次数

    /* 最近一次状态数据头 */
    uint16_t     last_total_points;      // 总点数
    uint16_t     last_sample_rate;       // 采样率
    float        last_health_score;      // 健康度
    uint32_t     last_status_timestamp;  // 状态时间戳
    FaultType_t  last_status_fault;      // 状态中携带的故障类型
} MasterNodeInfo_t;
```

### 4.9 RAM 下载缓冲区 (`MasterDownloadBuf_t`) — `app/task/master.h`

接收节点数据时累存采样点，满后整体写入 Flash。**全局静态变量，不在任务栈上**：

```c
typedef struct {
    uint8_t  active;              /* 是否有正在进行的下载 0=无 1=有 */
    uint8_t  node_id;             /* 当前接收的节点 */
    uint8_t  data_type;           /* 当前数据类型 */
    uint16_t expected_points;     /* 期望的总点数 */
    uint16_t received_points;     /* 已接收点数 */
    uint32_t sample_rate;         /* 采样率 */
    uint8_t  severity;            /* 故障级别 */
    uint8_t  flash_save_pending;  /* 延迟保存标志: 1=待写入Flash */

    uint8_t  recv_started;        /* recv任务: 已收到当前命令的首个header包 */
    uint16_t recv_raw_points;     /* recv任务: 帧解析后累计的原始数据点数 */
    uint16_t recv_expected_points;/* recv任务: 预期总点数 */

    NodeSample_t node_buffer[MASTER_NODE_UPLOAD_POINTS];  /* 采样点缓冲区 */
} MasterDownloadBuf_t;
```

## 5. Flash 存储布局

状态数据和波形数据**物理隔离**，各自占据 64KB Flash 区域，擦除操作互不影响。

```
0x08000000 ┌────────────────────────────┐
           │  程序代码 (192KB)          │
0x08030000 ├────────────────────────────┤  ← MASTER_STATUS_FLASH_BASE
           │  Node0 状态 [20KB]         │
           │  Node1 状态 [20KB]         │
           │  Node2 状态 [20KB]         │
           │  共 64KB                   │
0x08040000 ├────────────────────────────┤  ← MASTER_WAVE_FLASH_BASE
           │  Node0 波形 [20KB]         │
           │  Node1 波形 [20KB]         │
           │  Node2 波形 [20KB]         │
           │  共 64KB                   │
0x08050000 └────────────────────────────┘
```

| 区域 | 基址 | 每节点 | 总容量 | 用途 |
|------|------|--------|--------|------|
| 状态数据区 | `0x08030000` | `0x5000` (20KB) | 64KB | 存储 `NodeSample_t` 状态数据 |
| 波形数据区 | `0x08040000` | `0x5000` (20KB) | 64KB | 存储 `int16_t[]` 波形采样 |

**地址计算公式**:
```c
// 状态数据
addr = MASTER_STATUS_FLASH_BASE + node_id * MASTER_FLASH_PER_NODE

// 波形数据
addr = MASTER_WAVE_FLASH_BASE + node_id * MASTER_FLASH_PER_NODE
```

**Flash 读写接口** (`app/task/master_sys.c`):

| 函数 | 说明 |
|------|------|
| `master_flash_save_node_data(node, data, count)` | 保存 10 周期状态 → Flash |
| `master_flash_load_node_data(node, buf, max)` | 从 Flash 读取状态 → RAM |
| `master_flash_erase_node(node)` | 擦除节点状态区 |
| `master_flash_save_wave_data(node, data, len, offset)` | 追加保存波形 → Flash |
| `master_flash_load_wave_data(node, buf, len)` | 从 Flash 读取波形 → RAM |
| `master_flash_erase_wave(node)` | 擦除节点波形区 |

## 6. 通信流程

### 6.1 终端周期上传状态数据

```
终端                                   主控器
  │                                      │
  │── STATUS(NodeUploadData_t, 11B) ──→│ master_process: process_status_header()
  │  data_type=0x01, severity=NORMAL    │   → 记录 node_id, total_points
  │  node_index=0, total_points=40      │   → dl->expected_points = total_points
  │←── ACK ────────────────────────────│   → dl->active = 1
  │                                      │
  │── NODE_RAW(0x04) × N 包 ───────────→│ process_node_raw()
  │  每包 8 × NodeSample_t(128B)        │   → 累积到 dl->node_buffer[]
  │←── ACK (每包) ─────────────────────│
  │                                      │
  │                          收满预期点数 → master_flash_save_node_data()
  │                              → Flash 写入完成
```

### 6.2 故障触发 → 请求波形上传

```
终端                                   主控器
  │                                      │
  │── STATUS(FaultUploadHeader_t,15B)──→│ master_process: process_status_header()
  │  severity=WARNING, type=暂降         │   → 更新 node->severity, node->fault_type
  │←── ACK ────────────────────────────│   → 标记 node->fault_pending = 1
  │                                      │
  │                              ┌───────┴──────────┐
  │                              │  master_judge:   │
  │                              │  检测 severity≥   │
  │                              │  WARNING, 发送    │
  │                              │  CMD_REQUEST_WAVEFORM│
  │                              │  到终端            │
  │                              └───────┬──────────┘
  │                                      │
  │←── CMD_REQUEST_WAVEFORM ────────────│ send_lora_cmd()
  │    (0x10) 加密包                     │  加密 → LoRa → 等 ACK
  │──→ ACK ─────────────────────────────│
  │                                      │
  │── WAVE(WaveChunkHeader_t, 12B) ──→│ master_process: process_wave_header()
  │←── ACK ────────────────────────────│   → 擦除旧波形 Flash, state=RECV_WAVE
  │                                      │
  │── FLASH_WAVE(0x05) × N 包 ────────→│ process_flash_wave()
  │  每包 64 × int16_t(128B)            │   → 逐包追加写 Flash 波形区
  │←── ACK (每包) ─────────────────────│
  │                                      │
  │                          收满 sample_count → Flash 写入完成
```

| `MAX_ENCRYPT_DATA_LEN` | 128 | `chaos_encrypt.h` | 加密数据最大长度 |
| `LORA_BUFFER_SIZE` | 256 | `tasks.h` | LoRa 接收缓冲区 |
| `RECV_QUEUE_LENGTH` | 16 | `tasks.h` | 接收队列长度 |

### 任务优先级

| 任务 | 优先级 | 栈(字) | 说明 |
|------|--------|--------|------|
| `master_recv_task` | 5 | 256 | 接收任务（最高优先级防丢帧） |
| `master_process_task` | 4 | 512 | 解密+数据处理分发 |
| `master_judge_task` | 3 | 256 | 故障判断任务 |
| `master_poll_task` | 2 | 512 | 轮询调度任务 |

| 设备 | LoRa 地址 |
|------|----------|
| 主控器 | `0x000A` |
| 终端 0~2 | `0x000B ~ 0x000D` |

## 8. 指令码速查

| 指令码 | 名称 | 方向 | 说明 |
|--------|------|------|------|
| `0x10` | `CMD_REQUEST_WAVEFORM` | 主控→终端 | 请求上传指定节点的已录波形 |
