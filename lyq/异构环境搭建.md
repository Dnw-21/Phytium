## 三、详细步骤

### 步骤 1: 修改设备树添加 AMP 节点

PIOS 2.2 镜像的内核已内置 `CONFIG_HOMO_REMOTEPROC=y`, 但设备树缺少 AMP 节点, 需手动添加。

#### 1.1 提取当前设备树

fitImage 位于 SD 卡 4MB 偏移处, 先用 dumpimage 提取内核和 dtb:

```bash
# 从 SD 卡提取 (在板子上执行)
sudo dd if=/dev/mmcblk0 bs=1M skip=4 count=12 > ~/fitImage

# 传回开发机
scp user@192.168.88.10:~/fitImage .

# 拆出内核和 dtb
dumpimage -T flat_dt -p 0 -o kernel.gz fitImage
dumpimage -T flat_dt -p 1 -o board.dtb fitImage

# 反编译 dtb 为 dts
dtc -I dtb -O dts board.dts > board.dts
```

#### 1.2 添加 AMP 节点

在 `board.dts` 根节点 `/` 内, `memory@0` 之前插入:

```dts
reserved-memory {
    #address-cells = <0x02>;
    #size-cells = <0x02>;
    ranges;
    rproc: rproc@b0100000 {
        no-map;
        reg = <0x0 0xb0100000 0x0 0x19900000>;
    };
};

homo_rproc: homo_rproc@0 {
    compatible = "homo,rproc";
    status = "okay";
    homo_core0: homo_core0@b0100000 {
        compatible = "homo,rproc-core";
        remote-processor = <3>;
        inter-processor-interrupt = <9>;
        memory-region = <&rproc>;
        firmware-name = "openamp_core0.elf";
    };
};
```

**说明**:
- `remote-processor = <3>` 设备树写 CPU3, 实测 FreeRTOS 运行在 CPU1 (big核)
- `inter-processor-interrupt = <9>` 核间中断号 SGI #9
- `0xB0100000` 预留 409MB 给 OpenAMP 共享内存

#### 1.3 编译新 dtb

```bash
dtc -I dts -O dtb board.dts > board_amp.dtb
```

#### 1.4 打包新 fitImage

创建 ITS 文件 `new_fit.its`:

```dts
/dts-v1/;

/ {
    description = "U-Boot fitImage for Phytium Phytiumpi";
    #address-cells = <1>;

    images {
        kernel {
            description = "Linux kernel";
            data = /incbin/("kernel.gz");
            type = "kernel";
            arch = "arm64";
            os = "linux";
            compression = "gzip";
            load = <0x80080000>;
            entry = <0x80080000>;
            hash-1 { algo = "sha1"; };
        };
        fdt_1 {
            description = "FDT phytiumpi";
            data = /incbin/("board_amp.dtb");
            type = "flat_dt";
            arch = "arm64";
            compression = "none";
            hash-1 { algo = "sha1"; };
        };
    };

    configurations {
        default = "phytium@cecport";
        phytium@cecport {
            description = "phytimpi";
            kernel = "kernel";
            fdt = "fdt_1";
        };
    };
};
```

> ⚠️ **关键**: `default` 配置名必须是 `"phytium@cecport"`, 不能修改! U-Boot 按此名称查找配置。 这是踩过的最大坑——改了配置名会导致 U-Boot 报 "Could not find configuration node"。

```bash
mkimage -f new_fit.its new_fitImage
```

#### 1.5 刷入开发板

PIOS 2.2 自带 `runtime_replace_bootloader.sh` 脚本, 安全替换 SD 卡上的 fitImage:

```bash
scp new_fitImage user@192.168.88.10:~/fitImage
ssh user@192.168.88.10 "sudo runtime_replace_bootloader.sh fitImage && sudo reboot"
```

### 步骤 2: 验证 AMP 环境

重启后检查:

```bash
# 应看到 remoteproc0, 状态为 offline
ls /sys/class/remoteproc/
cat /sys/class/remoteproc/remoteproc0/state   # 输出: offline

# 确认 DT 节点存在
ls /proc/device-tree/reserved-memory/         # 应有 rproc@b0100000
```

### 步骤 3: 部署 FreeRTOS 固件

```bash
# 编译固件 (开发机上)
cd /home/alientek/Phytium_syscode/phytium-free-rtos-sdk-master/example/system/amp/openamp_for_linux
export AARCH64_CROSS_PATH="/home/alientek/Phytium_syscode/GCC编译器/arm-gnu-toolchain-13.3.rel1-x86_64-aarch64-none-elf"
make config_pe2204_phytiumpi_aarch64
make clean && make all
# 输出: pe2204_aarch64_phytiumpi_openamp_for_linux.elf

# 部署到开发板
scp pe2204_aarch64_phytiumpi_openamp_for_linux.elf user@192.168.88.10:/tmp/openamp_core0.elf
ssh user@192.168.88.10 "sudo cp /tmp/openamp_core0.elf /lib/firmware/ && sudo chmod 644 /lib/firmware/openamp_core0.elf"
```

### 步骤 4: 启动 FreeRTOS

```bash
# 启动从核
ssh user@192.168.88.10 "echo start | sudo tee /sys/class/remoteproc/remoteproc0/state"

# 验证
ssh user@192.168.88.10 "cat /sys/class/remoteproc/remoteproc0/state"
# 输出: running

# 停止
ssh user@192.168.88.10 "echo stop | sudo tee /sys/class/remoteproc/remoteproc0/state"
```

### 步骤 5: 编译 trace_reader 查看 FreeRTOS 输出

FreeRTOS 通过共享内存 (`0xC8000000`) 输出日志, Linux 端用 trace_reader 读取:

```bash
# 在开发板上编译 trace_reader
cat > /tmp/trace_reader.c << 'EOF'
#include <stdio.h>
#include <stdlib.h>
#include <fcntl.h>
#include <unistd.h>
#include <sys/mman.h>
#include <stdint.h>
#define BASE 0xC8000000
#define BSZ (1048576)
typedef struct { volatile uint32_t wi; volatile uint32_t ri;
  volatile uint32_t ov; char buf[BSZ-12]; } TB;
int main(void) {
  int fd=open("/dev/mem",O_RDWR|O_SYNC);
  if(fd<0){perror("open");return 1;}
  volatile TB*tb=(volatile TB*)mmap(NULL,BSZ,PROT_READ|PROT_WRITE,MAP_SHARED,fd,BASE);
  if(tb==MAP_FAILED){perror("mmap");return 1;}
  uint32_t lr=tb->ri;
  while(1){uint32_t wr=tb->wi;
    while(lr!=wr){putchar(tb->buf[lr]);lr=(lr+1)%(BSZ-12);}
    fflush(stdout);usleep(50000);
  }
}
EOF
gcc -Wall -O2 -o /home/user/trace_reader /tmp/trace_reader.c

# 运行查看
sudo /home/user/trace_reader
```

期望输出:
```
=== FreeRTOS LoRa v7-S2 (rpmsg-pipe) ===
Start sched
[SIM] init: 2-gen/5-bus, dt=0.0005s, 360000 steps
[SIM39] init: 10-gen/39-bus
[SIM9] init: 3-gen/9-bus
[SIM] waiting for Linux bind...
```

## 四、关键踩坑记录

### 坑1: ITS 配置名必须匹配 U-Boot

**现象**: U-Boot 报 `Could not find configuration node`, 系统无法启动。
**原因**: ITS 文件中 `default` 和配置节点名必须是 `"phytium@cecport"`, 不能随意修改。
**解决**: 保持配置名与原版 fitImage 完全一致。

### 坑2: dumpimage 和 mkimage 的 hash 格式

**现象**: 旧版 mkimage 不支持某些 hash 语法。
**原因**: PIOS 2.2 自带新版 u-boot-tools, hash 用 `hash-1 { algo = "sha1"; }` 格式。
**解决**: 遵循原版 fitImage 的格式, 不要用 `hash { algo = "sha256"; }` 等新版语法。

### 坑3: trace_reader 不存在于 PIOS 2.2 镜像

**现象**: 新烧写的 PIOS 2.2 镜像没有 `/home/user/trace_reader`。
**解决**: 需要手动编译(见步骤 5), 源码来自 DEBUG_GUIDE.md。

### 坑4: remoteproc stop/start 后 SHM 状态不重置

**现象**: FreeRTOS 重启后共享内存残留旧数据。
**原因**: remoteproc stop/start 不会清空共享内存。
**解决**: 重启后 SHM init 会重置 WI/RI/count 字段, 但数据区可能有旧内容。
