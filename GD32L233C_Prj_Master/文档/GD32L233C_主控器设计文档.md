# GD32L233C 主控器（Master Controller）设计文档

## 1. 概述

本文档描述基于 **GD32L233C + FreeRTOS** 的主控器软件设计。主控器通过 LoRa 无线模块（ATK-MWCC68D）与最多 **10 个终端节点**通信，接收终端上报的 **6 种数据类型**，将 **10 周期状态数据（400 点）** 和 **故障波形数据**分别存储到芯片内部 Flash 的不同区域，并根据节点故障状态向终端下发指令。

- **接收**: 终端上报的 6 种数据类型（状态头 / 波形头 / 电源 / 节点原始数据 / 波形原始数据 / 故障列表）
- **存储**: 10 周期状态数据（400 点 × 16B = 6400B）存于 Flash 状态区，波形数据（int16[]）存于 Flash 波形区
- **指令下发**: 混沌加密后通过 LoRa 发送，带 ACK 确认 + 最多 3 次重发
- **采集指令**: 采样率 6000Hz / 15000Hz、最大 500ms 波形采集指令

### 2.1 中断与 RTOS 优先级分析

| 组件 | NVIC 优先级 | 说明 |
|------|-----------|------|
| **USART1 RX 中断** | 3 (`nvic_irq_enable(USART1_IRQn, 3)`) | LoRa 模块接收, **每字节触发**, 仅写环形缓冲, 不调用 FreeRTOS API |
| FreeRTOS 临界区上限 | 2 (`configLIBRARY_MAX_SYSCALL_INTERRUPT_PRIORITY=2`) | `taskENTER_CRITICAL()` 只屏蔽 ≤2 的中断 |
| USART1 ISR 能否打断临界区 | **是** (3 > 2) | LoRa 数据不会因临界区而丢字节 |
| USART1 ISR 能否调 FreeRTOS API | **否** (3 > 2) | ISR 中仅操作 `usart1RxBuffer[]` 环形缓冲, 安全 |

## 3. 任务说明

### 3.1 master_recv_task — 主控接收任务

**优先级**: 4 | **堆栈**: 512 字 | **文件**: `app/task/master_recv.c`

从 LoRa 模块的 **USART1 环形缓冲**轮询接收终端发来的数据帧，经过 **帧解析（parse_frame）→ 混沌解密** 两步处理后，根据数据类型分发到对应处理器。

**接收—解密流水线**:

```
USART1 ISR (优先级3, 每字节触发)
   │  usart1RxBuffer[] ← usart_data_receive(USART1)  只写缓冲, 不调RTOS API
   ▼
环形缓冲 (mwcc68_uart.c)
   │
   ▼  usart1_data_available() 检查字节数
usart1_read_data(raw_buf, 256)  读出整帧原始字节
   │
   ▼  parse_frame() [master_recv.c 内置, CRC8校验]
   │  ① 扫描 0xAA 0x55 帧头
   │  ② 读 2B 大端长度 → data_len
   │  ③ CRC8 校验 data段 (多项式 0x07)
   │  ④ 校验帧尾 0x55 0xAA
   │  ⑤ 输出 inner_data = [sync_code 4B][data_type 1B][encrypted nB]
   ▼
chaos_decrypt_packet(&inner_data[5], enc_len, dec_buf, sync_code)
   │
   ▼
打印解密数据 → switch(data_type) 分发到 process_xxx()
```

**接收数据类型及处理方式**:

| 类型 | 枚举值 | 载荷格式 | 处理方式 |
|------|--------|---------|---------|
| `DATA_TYPE_STATUS` | 0x01 | `NodeUploadData_t`(11B) 或 `FaultUploadHeader_t`(15B) | 区分周期/故障上传, 更新节点快照, 标记 is_online |
| `DATA_TYPE_WAVE` | 0x02 | `WaveChunkHeader_t`(12B) | 擦除该节点波形 Flash, 状态进入累存 |
| `DATA_TYPE_POWER` | 0x03 | 预留 | 仅日志记录 |
| `DATA_TYPE_NODE_RAW` | 0x04 | `NodeSample_t` × N (每包 128B / 8 样本) | 累存到 `dl->node_buffer[]`, 满 400 点整块写 Flash |
| `DATA_TYPE_FLASH_WAVE` | 0x05 | `int16_t` × N (每包 128B / 64 样本) | 逐包追加写 Flash 波形区 |
| `DATA_TYPE_FAULT_LIST` | 0x06 | `uint8_t[8]` 故障有效性字节 | 统计有效故障数, 日志输出 |

**执行流程 (伪代码)**:
```
master_recv_task:
  while(1):
    avail = usart1_data_available()
    if avail == 0: vTaskDelay(10ms); continue

    raw_len = usart1_read_data(raw_buf, 270)
    printf 原始帧 hex

    if !parse_frame(raw_buf, raw_len,
                    inner_data, &inner_len, &inner_type, &timestamp):
        printf "[no valid frame]"; continue

    sync_code = inner_data[0..3]  (大端)
    rx_type   = inner_data[4]
    enc_len   = inner_len - 5

    解密 &inner_data[5] → dec_buf
    printf 解密后 hex

    识别 node_id
    switch(rx_type):
      DATA_TYPE_STATUS   → process_status_header()
      DATA_TYPE_WAVE     → process_wave_header()
      DATA_TYPE_NODE_RAW → process_node_raw()
      DATA_TYPE_FLASH_WAVE→ process_flash_wave()
      DATA_TYPE_FAULT_LIST→ process_fault_list()
```

### 3.2 master_judge_task — 主控故障判断任务

**优先级**: 5（最高） | **堆栈**: 256 字 | **文件**: `app/task/master_judge.c`

**周期**: 1000ms（`vTaskDelayUntil` 精确周期，每秒遍历所有 10 个节点）

**功能**:
- **离线检测**: 节点超过 15 秒无数据 → 标记 `is_online=0`
- **故障触发**: 节点在线且 `severity >= SEVERITY_WARNING`, `fault_type != FAULT_NONE`, `wave_pending == 0` → 向命令队列发送 `MASTER_CMD_REQ_WAVE`

**执行流程**:
```
while(1):
    vTaskDelayUntil(last_wake, 1000ms)
    for i in 0..9:
        n = g_nodes[i]
        if 超时15s: 标记离线, continue
        if !在线或wave_pending或fault_type==NONE: continue
        if severity < WARNING: continue

        cmd = {MASTER_CMD_REQ_WAVE, node_id=i, rate=6000, dur=250}
        xQueueSend(g_master_cmd_queue, &cmd, 0)
        n->wave_pending = 1
```

### 3.3 master_cmd_task — 主控指令下发任务

**优先级**: 3（最低） | **堆栈**: 256 字 | **文件**: `app/task/master_cmd.c`

从内部命令队列 `g_master_cmd_queue` 取出指令，通过 `send_lora_cmd()` 完成 **混沌加密 → LoRa_SendData_Direct 发送 → LoRa_ReceiveData 等 ACK → 最多 3 次重发**.

**指令包头格式** (`send_lora_cmd` 构建):
```
g_lora_pkt = [sync_code 4B, 大端][0x00 1B][encrypted_payload nB]
              ↑ 终端据此恢复混沌状态  ↑ 固定 0x00   ↑ chaos_encrypt_packet 输出
```

**支持指令**:

| 内部指令 | 终端码 | 参数 | 说明 |
|---------|--------|------|------|
| `MASTER_CMD_REQ_WAVE` | `CMD_REQUEST_WAVEFORM` (0x10) | node_id(1B) + fault_idx(1B) | 请求终端上传指定故障的已录波形 |
| `MASTER_CMD_REQ_FAULT_LIST` | `CMD_REQUEST_FAULT_LIST` (0x11) | 无 | 请求终端上传故障记录列表 |
| `MASTER_CMD_CLEAR_FLASH` | `CMD_CLEAR_FLASH` (0x12) | 无 | 清除终端 Flash 波形存储区 |
| `MASTER_CMD_WAVE_COLLECT` | `CMD_START_WAVE_COLLECT` (0x13) | sample_rate(2B) | **预留**: 指令终端启动新波形采集 |

**send_lora_cmd 发送流程**:
```
send_lora_cmd(node_id, cmd_code, params, param_len):
    payload[0] = cmd_code
    memcpy(&payload[1], params, param_len)

    chaos_encrypt_packet(payload, 1+param_len, g_enc_buf, &sync)
    g_lora_pkt = [sync(4B)][0x00][g_enc_buf]

    dest_addr = 0x0001 + node_id
    for retry in 0..2:
        LoRa_SendData_Direct(g_lora_pkt, 5+enc_len, dest_addr, LORA_CHN)
        LoRa_ReceiveData(ack_buf, &ack_len, 2000ms)  // 等ACK
        if OK:
            n->wave_pending = 0, n->cmd_retry = 0
            return
        log_warn("retry=%d", retry)

    log_warn("cmd failed after 3 retries")
```

## 5. Flash 存储布局

状态数据和波形数据**物理隔离**，各自占据 64KB Flash 区域，擦除操作互不影响。

```
0x08000000 ┌────────────────────────────┐
           │  程序代码 (128KB)          │
0x08020000 ├────────────────────────────┤  ← 终端侧波形区 (128KB, 终端使用)
           │  (主控不使用此区域)         │
0x08030000 ├────────────────────────────┤  ← MASTER_STATUS_FLASH_BASE
           │  Node0 状态 [6400B]        │
           │  Node1 状态 [6400B]        │
           │  ...                       │
           │  Node9 状态 [6400B]        │
           │  共 64KB                   │
0x08040000 ├────────────────────────────┤  ← MASTER_WAVE_FLASH_BASE
           │  Node0 波形 [~6400B]       │
           │  Node1 波形 [~6400B]       │
           │  ...                       │
           │  Node9 波形 [~6400B]       │
           │  共 64KB                   │
0x08050000 └────────────────────────────┘
```

| 区域 | 基址 | 每节点 | 总容量 | 用途 |
|------|------|--------|--------|------|
| 状态数据区 | `0x08030000` | `0x1900` (6400B) | 64KB | 存储 400 点 `NodeSample_t`（10 周期 × 40 点/周期） |
| 波形数据区 | `0x08040000` | ~6400B（可变） | 64KB | 存储 `int16_t[]` 波形采样（6kHz×250ms=1500点 或 12kHz×400ms=4800点） |

**地址计算公式**:
```c
// 状态数据
addr = MASTER_STATUS_FLASH_BASE + node_id * MASTER_FLASH_PER_NODE

// 波形数据
addr = MASTER_WAVE_FLASH_BASE + node_id * MASTER_FLASH_PER_NODE
```

**Flash 读写接口** (`app/task/master_sys.c`):

| 函数 | 说明 |
|------|------|
| `master_flash_save_node_data(node, data, count)` | 保存 10 周期状态 → Flash |
| `master_flash_load_node_data(node, buf, max)` | 从 Flash 读取状态 → RAM |
| `master_flash_erase_node(node)` | 擦除节点状态区 |
| `master_flash_save_wave_data(node, data, len, offset)` | 追加保存波形 → Flash |
| `master_flash_load_wave_data(node, buf, len)` | 从 Flash 读取波形 → RAM |
| `master_flash_erase_wave(node)` | 擦除节点波形区 |

## 6. 通信流程（重点）

### 6.1 LoRa 空中帧格式

终端→主控所有数据包遵循统一的 LoRa 空中帧格式（由终端 `data_frame.c` 的 `send_node_data_with_ack()` 构建）：

```
┌──────────┬───────────┬────────────────────────────────────────────────┬──────┬──────────┐
│ 0xAA     │ data_len  │              data段                             │ CRC8 │ 0x55     │
│ 0x55     │ (2B, 大端)│           (data_len 字节)                       │ (1B) │ 0xAA     │
├──────────┼───────────┼────────┬──────────┬──────────┬─────────────────┼──────┼──────────┤
│ 帧头  2B │ 长度    2B│  ts 4B │ d_type 1B│ sync  4B │ d_type 1B       │校验 1B│ 帧尾  2B │
│          │           │        │  (外层)  │          │  (内层,重复)    │      │          │
│          │           │        │          │          │ payload nB       │      │          │
└──────────┴───────────┴────────┴──────────┴──────────┴─────────────────┴──────┴──────────┘
```

**帧开销**: 7 字节 (2帧头 + 2长度 + 1CRC + 2帧尾)

**字段说明**:

| 字段 | 偏移 | 大小 | 说明 |
|------|------|------|------|
| 帧头 | 0 | 2B | 固定 `0xAA 0x55` |
| 长度 | 2 | 2B | data段 字节数，**大端序**。`len = 10 + n` (ts 4 + d_type 1 + sync 4 + d_type 1 + payload n) |
| timestamp | 4 | 4B | 发送时间戳 (ms)，大端序 |
| data_type(外) | 8 | 1B | 数据类型 (0x01~0x06) |
| sync_code | 9 | 4B | 混沌加密同步码，大端序。为 0 时表示明文 |
| data_type(内) | 13 | 1B | 数据类型 (与外层相同)，属于加密文件的一部分 |
| payload | 14 | nB | 混沌加密后的应用层载荷 |
| CRC8 | 4+len | 1B | 对 data段 的 CRC-8 校验 (多项式 `0x07`) |
| 帧尾 | 5+len | 2B | 固定 `0x55 0xAA` |

**CRC8 多项式**: `0x07` (x⁸ + x² + x + 1)

### 6.2 主控接收—解密流水线

主控 `master_recv_task` 采用 **中断接收 + 任务解析** 的二级流水线：

```
┌─ 中断层 (USART1 ISR, 优先级 3) ─────────────────────────────┐
│ 每字节触发: usart_data_receive(USART1) → usart1RxBuffer[]    │
│ 只写环形缓冲, 不调 FreeRTOS API, 耗时 ~3μs/字节              │
│ 优先级 3 > FreeRTOS临界区上限 2 → 临界区内也不会丢数据        │
└───────────────────┬──────────────────────────────────────────┘
                    │ 环形缓冲填有数据
                    ▼
┌─ 任务层 (master_recv_task, 优先级 4) ────────────────────────┐
│ while(1):                                                     │
│   usart1_data_available() → 检查字节数                        │
│   if 无数据: vTaskDelay(10ms); continue                       │
│                                                               │
│   usart1_read_data(raw_buf) → 读出原始帧字节                  │
│                                                               │
│   parse_frame(raw_buf) ──┐                                    │
│     ① 扫描 0xAA 0x55    │  帧解析 (从终端 data_frame.c 移植) │
│     ② 读 2B 大端长度   │                                     │
│     ③ CRC8 校验 data段 │                                     │
│     ④ 校验帧尾 0x55 0xAA│                                    │
│     ⑤ 输出 inner_data  │                                     │
│   ◄────────────────────┘                                     │
│                                                               │
│   inner_data = [sync(4B)][type(1B)][encrypted(nB)]            │
│                                                               │
│   chaos_decrypt_packet(&inner_data[5], enc_len, dec_buf,      │
│                         sync_code)                             │
│   → 同步码恢复混沌状态 → 异或解密 → 置换逆操作 → 明文         │
│                                                               │
│   printf 解密后hex → switch(type) 分发处理                    │
└───────────────────────────────────────────────────────────────┘
```

**关键设计点**:

| 设计点 | 说明 |
|--------|------|
| `parse_frame()` 内置 | 从终端 `data_frame.c` 原样复制 `calc_frame_crc8()` + `parse_frame()`，不依赖主控的 `data_frame.c`（主控侧此文件功能不同） |
| CRC8 校验 | 严格校验 data段 CRC8，错误帧直接丢弃，防止解密乱码 |
| 大端序 | 所有多字节字段（长度、时间戳、sync_code）均为大端序，与终端一致 |
| 明文回退 | `sync_code == 0` 时跳过解密，直接使用原始字节 |

### 6.4.1 主控下发指令 — ACK 重发机制

主控指令下发采用 **加密 + 定向发送 + ACK等待 + 3次重发** 的可靠传输机制：

```
send_lora_cmd(node_id, cmd_code, params, param_len):
  ┌─────────────────────────────────────────┐
  │ ① 构建 payload = [cmd_code | params]     │
  │ ② chaos_encrypt_packet(payload)          │
  │    → g_enc_buf + sync_code              │
  │ ③ 包头: [sync(4B,BE)][0x00][g_enc_buf] │
  │                                         │
  │ dest_addr = SLAVE_ADDR_BASE + node_id   │
  │                                         │
  │ for retry = 0, 1, 2:                   │
  │   ④ LoRa_SendData_Direct(              │
  │        g_lora_pkt, 5+enc_len,          │
  │        dest_addr, LORA_CHN)            │
  │   ⑤ LoRa_ReceiveData(ack_buf,          │
  │        &ack_len, 2000ms)               │
  │      → OK: wave_pending=0, return      │
  │      → FAIL: retry++                   │
  │                                         │
  │ ⑥ 3次失败 → log_warn                   │
  └─────────────────────────────────────────┘
```

| 参数 | 值 | 说明 |
|------|-----|------|
| ACK 超时 | 2000ms | `LoRa_ReceiveData` 超时等待 |
| 最大重试 | 3 (`MASTER_CMD_RETRY_MAX`) | 3 次无应答判定失败 |
| 定向发送 | `LoRa_SendData_Direct` | 直接指定目标地址 `0x0001+node_id` 和信道 23 |
