# 飞腾 PE2204 OpenAMP 设备树覆盖层解析

这份是 **phytium PE2204（飞腾 CEK8903）异构双核 OpenAMP 通信** 的 DTBO 设备树插件，作用：预留远端核共享内存 + 注册本地 Linux 侧 rproc 驱动节点，实现 Linux ↔ FreeRTOS 双域 RPMsg 通信。

## 1. 头部标识

```
/dts-v1/;
/plugin/;
```

- `/dts-v1/`：DTS 语法版本 1
- `/plugin/`：标记为**设备树 Overlay 插件**，动态加载到主设备树，不修改原厂 dts

## 2. 顶层兼容匹配

```
/ {
    compatible = "phytium,pe2204";
```

限定该 overlay 仅加载到 PE2204 平台，防止加载到其他飞腾芯片。

## 3. fragment@0 挂载根节点

```
fragment@0 {
    target-path = "/"; // 挂载到设备树根目录
    __overlay__ {
        // 新增节点
    };
};
```

`target-path="/"` 代表把下方所有节点追加到系统根节点下。

## 4. reserved-memory：远端核共享内存区

```
reserved-memory {
    #address-cells = <2>;
    #size-cells = <2>;
    ranges;

    rproc@b0100000 {
        no-map;
        reg = <0x0 0xb0100000 0x0 0x19900000>;
        phandle = <100>;
    };
};
```

### 关键字段说明

1. `#address-cells = <2>` / `#size-cells = <2>` 飞腾 64 位平台，地址 / 长度用**两个 32 位单元**表示（高 32bit + 低 32bit）

2. ```
   reg = <0x0 0xb0100000 0x0 0x19900000>
   ```

   - 起始物理地址：`0x00000000 b0100000`
   - 分配内存大小：`0x00000000 19900000` 换算：`0x19900000 = 425MB` 共享内存

3. `no-map` Linux 内核**不做线性地址映射**，仅保留物理内存预留，交给 rproc/OpenAMP 管理，避免 Linux 占用破坏远端固件。

4. `phandle = <100>` 句柄编号，下方`homo_rproc`通过`memory-region = <100>`引用这片内存。

## 5. homo_rproc：OpenAMP 远端处理器驱动节点

```
homo_rproc {
    compatible = "homo,rproc";
    remote-processor = <3>;
    inter-processor-interrupt = <9>;
    memory-region = <100>;
    firmware-name = "openamp_core0.elf";
    status = "okay";
};
```

### 逐字段解释

1. `compatible = "homo,rproc"` 匹配自定义 rproc 驱动（自研 OpenAMP 远端核管理驱动，非内核标准 ti,remoteproc）

2. `remote-processor = <3>` 指定远端核硬件编号 3，对应 PE2204 上运行 FreeRTOS 的从核

3. `inter-processor-interrupt = <9>` 核间中断门号 9：Linux 与从核互相收发消息的门铃中断（VirtIO ring 门铃）

4. ```
   memory-region = <100>
   ```

   关联上方

   ```
   rproc@b0100000
   ```

   预留内存，作为：

   - 从核固件加载区域
   - VirtIO vring 环形队列缓冲区
   - RPMsg 消息收发共享内存

5. `firmware-name = "openamp_core0.elf"` Linux 内核会从 `/lib/firmware/` 目录加载该 ELF 固件，解析段、加载到`0xb0100000`，上电启动从核

6. `status = "okay"` 节点使能，内核加载对应驱动

# 整体工作流程

1. Linux 启动加载本 DTBO overlay，在根节点创建预留内存 + rproc 设备节点
2. 内核解析`homo_rproc`，匹配`homo,rproc`驱动
3. 驱动读取 firmware-name，加载 openamp_core0.elf 到 0xb0100000 物理内存
4. 初始化 VirtIO 环形队列、核间中断 9
5. 启动远端 Core0（FreeRTOS），建立 RPMsg 通道实现异构核数据交互

# 常见调试点 & 修改建议

## 1. 内存大小 / 地址调整

如果固件加载报段溢出，修改 reg 第二个长度参数：

```
// 示例扩大到512MB
reg = <0x0 0xb0100000 0x0 0x20000000>;
```

## 2. 核号、中断不对

核对芯片手册 PE2204 从核编号、IPC 中断号，修改`remote-processor`/`inter-processor-interrupt`。

## 3. 固件找不到

把`openamp_core0.elf`放入开发板 `/lib/firmware/`，执行`depmod`更新固件索引。

## 4. no-map 作用

删除`no-map`会导致 Linux MMU 映射该段内存，Linux 业务会覆盖从核运行内存，死机 / 通信异常，**必须保留**。

## 5. Overlay 编译命令

```
dtc -I dts -O dtbo -o pe2204-openamp.dtbo pe2204-openamp-overlay.dts
```

加载方式（内核 config 开启 CONFIG_OF_OVERLAY）：

```
# 加载overlay
cat pe2204-openamp.dtbo > /sys/kernel/config/device-tree/overlays/rproc/dtbo
# 卸载
rmdir /sys/kernel/config/device-tree/overlays/rproc
```

# 潜在风险点

1. 预留内存`0xb0100000`不能和内核、外设、其他保留内存地址重叠，否则系统崩溃
2. 从核 FreeRTOS 工程链接脚本，运行地址必须匹配`0xb0100000`，否则上电跑飞
3. 核间中断号冲突会导致 RPMsg 收发卡死、无中断响应
